웹사이트 업데이트 = 새로운 서버를 만들어서 redirect하는 것.
- 즉, 서버의 코드는 영원한 것이 아님. 
- 기존의 서버는 그냥 없어지는 것.

=================
(주의) 사용자가 생성한 content는 별도의 서버에 저장되어야 함. 
- 사용자가 업로드한 file을 내 서버에 직접 저장하는 것은 bad practice.
- 보안 문제만이 아님. 
- 서버를 업데이트할 때 해당 file들도 전부 다 복제해야 하거나, 사라지게 됨.
- 너무 큰 파일을 업로드하는 사용자가 있다면, 서버 전체에 무리가 갈 수도 있음. 모든 사용자에게 해가 됨.

[Moltar 사용]
1) 사용자가 생성한 데이터들, 파일들은 Amazon Cloud에 업로드
2) Amazon Cloud에서 URL을 받아와서 저장.
